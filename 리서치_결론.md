## 1. 전체 구조

### 파이프라인

```
[소스 수집] → [팩트 선별] → [발화 생성] → [피드 게시]
                                ↑
                         [피드백 루프]
                     이전 발화의 답글 맥락
```

### 카테고리 구조

```
configs/
├── kbo.yaml                    ← 고정 (매일)
├── stock.yaml                  ← 고정 (매일)
├── broadcast_무명전설.yaml      ← 프로그램별 (주간)
├── broadcast_나는솔로.yaml      ← 프로그램별 (주간)
└── broadcast_[새프로그램].yaml   ← 운영자가 수동 추가
```

- KBO, 주식: 카테고리 = 봇 1개
- 방송: 프로그램 = 봇 1개. 종영 시 yaml 비활성화, 새 프로그램 추가
- 카테고리 추가 시: 크롤러 + 프롬프트(yaml) + 설정만 추가하면 됨

### 전/후 구조 (모든 카테고리 공통)

```
         KBO              주식              방송
[전]   경기 전 14:00    장 전 08:00      방영 전 오전
         ↓                ↓                ↓
       유저 답글          유저 답글         유저 답글
         ↓                ↓                ↓
[후]   경기 후 22:00    장 후 15:30      방영 후 23:00
       (전 답글 맥락     (전 답글 맥락     (전 답글 맥락
        + 결과 반영)      + 결과 반영)      + 결과 반영)
```

## 2. 구현 과제

### 과제 A: 프롬프트 강화 (바로 적용)

configs/kbo.yaml의 prompt 섹션을 아래 기준으로 업데이트:

**캐릭터 원칙:**
- 톤: 해설위원 70% / 흥분하는 팬 20% / 커뮤니티 언어 10%
- 핵심: Personality through selection — 의견 말하지 않고 논쟁적 팩트를 골라서 던짐
- 에너지: 이벤트 크기에 비례 (일상=차분, 활약=살짝 올림, 역사적=최대)
- 길이: 팩트 1줄 + 질문 1줄 (1-3문장)
- 금지: 가치판단, 초성체 남발, 청소년 슬랭(ㄹㅇ, 개~)
- 반말 유지. ~입니다(60%) + ~인데(25%) + ~죠(10%) + ~!(5%)
- ㅋㅋ 최대 1회/글

**포맷 로테이션 (같은 형식 반복 금지):**
1. 이항선택 ("A야 B야?") — 주 3-4회
2. 빈칸채우기 ("올 시즌 MVP는 ___") — 주 1-2회
3. 순위 도전 ("TOP 3 만들어봐") — 주 1회
4. 예측 ("오늘 결과 맞춰봐") — 경기/이벤트 전
5. 반론 유도 ("이거 반박 가능?") — 주 1회

**팩트 선별 기준 (3속성 중 2개 이상 해당하는 팩트 선택):**
- 의외성: 예상과 다른 사실 (순위 갭, 기록 갱신)
- 긴장감: 충돌하는 팩트 2개 (발언↔수치, 리드→역전)
- 관여도: 내가 한마디 하고 싶어지는 사실 (내 팀, 내 종목, 논쟁 진행 중)

**30-50대 남성 트리거:**
- 역대 기록/순위 ("역대 3번째", "시즌 최초")
- 세대 비교 ("박찬호 이후 최초")
- 상반된 팩트 병치 ("타율 1위인데 팀 순위 8위")
- 간결함 (장황하면 아마추어로 봄)
- 정시 발행 (경기 후 30분 이내)

### 과제 B: 피드백 루프 (1단계 후반, 답글 쌓인 후)

**파이프라인 변경:**
```
[AS-IS] 이벤트 수집 → 컨텍스트(뉴스) → 발화 생성
[TO-BE] 이벤트 수집 → 컨텍스트(뉴스 + 이전 발화의 답글) → 발화 생성
```

**구현:**
1. 직전 발화(같은 카테고리)의 답글을 공감순으로 3-5개 가져오기
2. writer_agent의 build_prompt에 `[이전 발화에 대한 유저 반응]` 섹션 추가
3. 규칙:
   - 유저 반응이 있으면 자연스럽게 흐름을 이어가되 새로운 팩트를 중심에
   - "어제 ~라는 의견이 있었는데" 식으로 직접 인용하지 말 것
   - 답글이 없으면 기존 방식 그대로

**예시:**
```
[전] "오늘 한화 선발 쿠싱 데뷔전. 트리플A 다승왕 출신인데, KBO 적응 바로 될까?"
  → 답글: "외인 선발 첫 등판은 원래 5이닝이면 성공" (공감 8)
  → 답글: "컨트롤 좋다던데 기대됨" (공감 5)

[후] "쿠싱 5이닝 2실점. 데뷔전치고 나쁘지 않은데, 한화 타선이 3득점밖에 못 해서 패배. 
     쿠싱 문제야 타선 문제야?"
```

### 과제 C: 주식 카테고리 추가

stock.yaml 생성. KBO와 동일한 파이프라인이지만 차이점:
- 톤: 더 차분 (해설위원 80% / 흥분 10% / 커뮤니티 10%)
- 주식은 돈이 걸려있어서 과한 캐릭터가 신뢰를 깎음
- 가치판단 금지가 더 엄격 ("삼성전자 사라" 절대 금지)
- 소스: 다음 금융 (코스피/코스닥) + 다음 뉴스 (경제)
- 타이밍: 장 전 08:00-09:00, 장 후 15:30-16:00

**논쟁적 팩트 유형 (주식 특화):**
- "실적 어닝서프라이즈인데 주가 하락"
- "증권사 목표가 상향 vs 실제 주가 하락"
- "3일 연속 하락인데 외국인은 매수 전환"
- "반도체 전멸인데 2차전지만 상승"

**문체:**
```
좋음: "코스피 2,700 깨졌다. 외국인 순매도 1,200억. 근데 기관은 오히려 담고 있는데, 누가 맞는 거야?"
나쁨: "삼성전자 주식을 사는 게 좋을 것 같습니다."
```

### 과제 D: 방송 카테고리 구조

프로그램별 yaml. broadcast.yaml이 아니라 broadcast_[프로그램명].yaml로 분리.

**프로그램 선정 기준:**
- 30-50 남성 시청층 + 매주 정기 방영 + 토론거리 + 뉴스에 기사 나옴

**KBO/주식과 다른 점:**
- 가장 캐주얼한 톤 (해설위원 60% / 흥분 25% / 커뮤니티 15%)
- ㅋㅋ 사용 허용 빈도 높음
- 소스: 다음 뉴스 검색 (연예) 프로그램명으로 검색
- 타이밍: 방영 전 당일 오전, 방영 후 30분~1시간

**논쟁적 팩트 유형 (방송 특화):**
- "심사위원 3:2로 갈린 탈락"
- "시청률 12%로 1위인데 전주보다 하락"
- "심사위원이 '역대 최고'라고 했는데 시청자 반응은?"

**문체:**
```
좋음: "무명전설 이번 회 탈락자 A씨. 심사위원 3:2로 갈렸는데, 솔직히 B씨보다 잘하지 않았어?"
나쁨: "A씨가 탈락해서 아쉽습니다."
```

## 3. 소스 개선 (완료)

crawler_agent.py에 이미 적용됨:
- `fetch_article_lead()` — 다음 뉴스 기사 본문에서 리드문(300자) 추출
- `crawl_news()` — 제목 + 리드문 함께 반환
- URL 중복 제거

소스 구조:
```
크롤러:
├── 공통: 다음 뉴스 검색 (헤드라인 + 리드문)
├── KBO: 다음 스포츠 (경기결과) — 기존
├── 주식: 다음 금융 (시황 + 종목) — 추가 필요
└── 방송: 다음 뉴스 (프로그램명 검색) — 기존
```

## 4. 설계 원칙 (리서치 근거)

| 원칙 | 근거 | 출처 |
|------|------|------|
| 봇은 원글만 쓰고 답글에 개입하지 않음 | 봇이 답글에 끼어들면 토론 품질 하락 | Notre Dame Reddit 14년 연구 |
| 봇에게 명확한 역할 부여 | 범용 봇(Clyde)·톱다운 봇(Meta AI)·역할 불명확 봇(Butterflies) 모두 종료/축소 | Discord, Meta, Butterflies 실패 사례 |
| 발화에 항상 질문 포함 | 질문형 포스트 댓글 86% 증가 | Reddit 데이터사이언스 2019 (2B+ 댓글 분석) |
| 이항선택 포맷 우선 | 이항선택 포맷이 답글 4배 | Buffer 1M+ 트윗 분석 2023 |
| 40-60 갈리는 팩트 선택 | 의견이 갈려야 양쪽 다 답글 씀. 90% 동의=조용, 50:50=최대 참여 | 사회심리학 negativity bias 연구 |
| 봇 댓글이 해당 글 참여 높임 | 관련성 있는 봇 댓글 → 댓글 +23%, 좋아요 +11% | INFORMS 웨이보 CommentRobot 연구 2025 |
| 봇만으로 전체 활성화 불가 | 개별 글 참여는 높이지만 플랫폼 전체 활동량은 변화 없음 | 같은 INFORMS 연구 |
| 피드백 루프는 블루오션 | 90개 사례 조사 결과 답글→다음 발화 반영 구조를 실현한 서비스 없음 | ai-research 전체 리서치 |
